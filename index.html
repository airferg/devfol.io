Mystery Organism

Overview

This was a small project apart of the CodeCademy Full Stack Engineer Course. I really enjoyed this one, because it was a new concept of working with objects and arrays in JavaScript. Here, there is a hypothetical new organism scientists are calling a pAequor. In this project, I created an object that randomly generates the dna strand of this organism, determines its survival rates, it's complimentary strand, and much more.

Functionality

To start, a function is made to randomly generate a random dna base - either Adenine [A], Theanin [T], Cytosine [C], or Guanine [G]. Next, another function calls the previous one and mocks up 15 random bases into one dna strand. This is the dna of the pAequor. Next comes the object. Each one given a specimen number, and a dna strand. In this object, functions are created to help research. The first function being one that mutates the dna bases into something different. For example, if a dna base in the dna strand is C, the function will mutate the original dna base in that strand to either an A, T, or G. Next function compares the dna of this pAequor to a new one. It randomly generates another dna base, and runs through each strand, comparing how many bases are the same in both. Additionally, there is a function that uses a hypothetical sickness to determine if a specimen will survive. This sickness can only be survived if the specimen is made up of atleast 60% C and G bases. So, it iterates through the dna strand, tallies how many C and G bases there are in the total, and determines a percentage. If the percentage is over 60%, the specimen will survive. Another function creates the complimentary strand of dna to the pAequor's dna strand. Iterating through each bases, it creates a new string and pushes the complimentary base to it. This creates the perfect complimentary dna strand. Finally, there is one last function oustide of the object. This one creates an array for 30 dna strands. However, the only strands allowed in are strands that will survive the sickness, and also the complimentary strands to each. So for this, the object must be called within the parameters of the new function, creating an array of the ultimate pAequor specimens and the strands that compliment them best.

Credit Card Checker

Overview

This project was apart of the CodeCademy Full Stack Engineer Course. Personally, this is my favorite one I have done. As my education is in finance, I have a love for financial technology, and solving financial problems in code. In this one, I was given an array of valid and invalid card numbers, and also an array of mystery numbers, in which I didn't know if they were valid or invalid. This project was broken down into multiple functions, and functions within functions.

Functionality

At the start, I am given 5 valid card numbers in an array. Next, I am given 5 invalid card numbers in an array. Also, I am given 5 mystery card number in an array, not sure which numbers are valid or invalid. Next, I am given a batch array, that contains all of the other arrays.
Now, the function to validate the card numbers begins. The function name is validateCred(). This function checks whether a given credit card number (represented as an array) is valid or not using the Luhn Algorithm. The function starts by initializing the decision variable, an empty array sub, and a count variable to track the number of digits processed. A for loop iterates over the array arr in reverse (from the last number to the first number). For the last element in the array (which is typically the check digit), it is added directly to the sub array without modification. For every other digit (based on the Luhn algorithm), if it's in an odd index from the end, it is doubled. If the doubled value is greater than 9, 9 is subtracted from it (this is part of the Luhn algorithm to prevent a number from being too large). If the index is not one of those that needs to be doubled, the number is simply pushed into the sub array. The function sums up all values in the sub array using the reduce() method. If the sum is divisible by 10 (sum % 10 === 0), the credit card number is valid, and decision is set to true. Otherwise, decision is false. The function returns the boolean decision, indicating whether the card is valid or not.
Next function, findInvalidCred(nestedArray). This function takes an array of credit card numbers (arrays) and returns only those that are invalid. The function initializes an empty array invalidCards. It loops over each card (array) in nestedArray. It uses the validateCred() function to check each card. If validateCred() returns false, the card is added to the invalidCards array. After checking all cards, it returns the array invalidCards, which contains only the invalid cards.
On to the next function, idInvalidCardCompanies(invalidArray). This function identifies the companies that issued the invalid credit cards. It starts by initializing an empty array called mailedOut and retrieves the invalid cards using findInvalidCred(). For each invalid card, it checks the first digit (nums[0]) to determine the company:

  • 3 indicates Amex (American Express).

  • 4 indicates Visa.

  • 5 indicates Mastercard.

  • 6 indicates Discover.

  • If none of these values match, it assigns "Company not found".

If the company has not already been recorded in mailedOut, it adds the company to the array. The function returns mailedOut, which contains a list of all the companies associated with invalid cards.

Beer Advice From Dad Generator

Overview

This was a goofy project I made. I wanted to make some type of generator in javascript. I thought about barstool sports, friday beers, old row, that kind of niche. All of those guys look the same. Hats, mustaches, sunglasses, long hair that curls out the back, college football, and most importantly, beer. So the idea hit me, let's make a beer advice generator from dad.

Functionality

First, I initialized three different arrays. The first one being the introduction, the second being the actual advice, the last part being the outro. Using Chatgpt, I found 30 different introductions, advices, and outros, all related to beer and dad jokes. For the function, it takes a number as a parameter. This is the number of phrases you want. Next, the function initializes an emtpy array. Next, it loops through the number of iterations passed into to the function before. During each iteration, an intro, advice, and outro phrase is chosen in their respective arrays using the Math library in javascript. After each part of the message is chosen, they are each pushed to a temporary array. Next, each part of the array is put together into one string by removing the '' and , . Next, this string is pushed to the final array. After each iteration, this final array will be filled up with beer advice messages from dad.

Mad Menace

An Overview

This was my fist solo project in code. It was created as the final exam project for Harvard's computer science program, CS50. The parameters of the project were simple. I was to make a web application using HTML, CSS, JavaScript, Python, and SQL. It was my first time creating both a back end and front end, and also working with a databases. My idea was to make a social media specifically for skaters, where they could share photos and videos with captions, and also schedule meet up events for people that want to skate together. It had an entire login system, validation, password hashes, user id and password database. I really enjoyed making it. All validation was through the backend in python, which was a first for me. The name of this social media was Mad Menace. When the user actually logs in, they are greeted by the home page. It is a collage of photos and videos from other users. It uses python to call for the most recently posted media from all users, searching through the media table in the SQL database. Each photo has the user id and a caption below. It is very similar to VSCO's format for their photos. Next, each person has a profile, where they can upload a profile photo, see their own media, and delete posts. There is a search engine function at the top, where you can search for other users. This was tricky, but eventually I was able to do it. As the user types, with each letter typed in, python searches through the database for users with similar usernames. Next, users are also given the ability to fill out a form and post a skate meet up. It was an HTML form that asked for a username, location, date, and time. Once posted, you could see all posts in the skate meetup discover section. This part just searched through the database and posted each meetup in an html table.
